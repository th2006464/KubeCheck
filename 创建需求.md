# 审批配置数据校验需求文档

​**适用人员**​：开发人员

​**源文件编码**​：CSV 统一使用 UTF-8

​**整体目标**​：对审批配置CSV数据进行规则校验，实现HTML列表页面展示 + 带标识列的CSV文件导出

​**技术架构**​：前端 Bootstrap 3（远端引用）、服务端 IIS + .NET 8

## 一、技术环境规范

### 前端技术

* UI 框架：**Bootstrap 3**
* 引用方式：​**远端CDN引用**​，不使用本地静态资源部署
* 页面载体：HTML 列表形式展示校验结果

### 服务端架构

* 运行环境：IIS
* 开发框架：.NET 8
* 数据来源：UTF-8 编码 CSV 文件

## 二、数据列结构定义

### 列分区规则

1. **条件列**

所有 `role` 系列列左侧的全部列，列数不固定。

多行条件列内容​**字面完全一致**​，划分为​**同一个条件组**​；列内`*`按普通字符匹配，不做通配符解析。

1. **审批人列**

列名格式：`role`、`role1`、`role2` ... `role20`

最大支持20列，不同CSV文件列数可动态变化；单元格空值代表无对应审批人。

### 导出文件新增列

* 新增位置：表格**最左侧**
* 列名称：存在异常值
* 单元格取值：是 / 否

## 三、核心校验规则

校验范围：​**仅在同一个条件组内执行校验**​，同一条件组审批配置要求完全统一。

任意命中以下任意一条规则，该行判定为异常。

### 规则1：审批人数不一致

* 统计口径：只统计审批人列​**非空单元格**​，空值不计入人数。
* 判定：同一条件组内，各行有效审批人数不统一（偏多/偏少）→ 异常。

### 规则2：审批人员内容不一致

同一条件组内，审批人列的人员姓名、排列顺序、空值分布存在差异 → 异常。

### 规则3：配置完全重复

同一条件组内，多行审批人列内容、顺序、空值完全一致（冗余重复配置）→ 异常。

## 四、前端 HTML 列表展示要求（Bootstrap 3）

1. 基于 Bootstrap 3 构建页面布局与表格/列表样式，框架采用​**远端CDN引用**​；
2. 完整展示全部原始数据行；
3. 标记为**异常**的行，通过 Bootstrap 内置样式做视觉区分（底色高亮、文字标色等），样式沿用 Bootstrap 3 原生类；
4. 可选功能：支持悬浮/附属区域展示该行具体异常原因。

## 五、CSV 文件下载导出要求

1. 文件格式：保持原 CSV 格式，编码 UTF-8；
2. 列顺序：`存在异常值`（首列） + 原始所有列（条件列 + 全部role审批人列）；
3. 字段规则：
   * 校验异常行：存在异常值 = 是
   * 校验正常行：存在异常值 = 否
4. 约束：原始所有单元格内容、列顺序、数据格式​**完整保留，不做修改**​。

## 六、异常原因说明（前端展示文案）

用于展示具体异常类型，文案参考：

1. 同一条件组内审批人数不一致
2. 同一条件组内审批人员/排列顺序不一致
3. 同一条件组内存在重复审批配置

## 七、通用技术约束

1. 自动分区：程序自动识别条件列、role审批人列，无需手动配置列数；
2. 动态兼容：自动适配 1~20 个 role 审批列，支持不同列数的CSV文件；
3. 字符规则：条件列中的 `*` 作为普通文本参与匹配，不启用通配逻辑；
4. 部署要求：项目部署在 IIS 中，基于 .NET 8 运行，适配对应运行环境。
